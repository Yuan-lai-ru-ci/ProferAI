# 团队工作区文件管理

> 阶段收尾：2026-06-20

团队工作区的文件管理已经进入可用状态：左侧是团队文件主视图，右侧是可收起的 Agent 面板。文件清单以团队服务器为准，本地 `workspace-files/` 只作为预览、打开和 Agent 解读时的缓存目录。

## 当前能力

- 团队文件支持文件 / 文件夹上传、远程清单展示、新建文件夹、删除、下载、预览、打开所在位置和路径复制。
- 上传文件后会同步写入本地 `workspace-files/` 缓存；自己上传的文件预览时优先使用这个本地路径。
- 如果本地缓存不存在，或者文件来自其他成员，预览、下载、拖入 Agent 时会先下载到 `workspace-files/` 再使用。
- 团队模式主界面使用文件管理顶栏内嵌窗口控制按钮，避免右侧 Agent 收起条和原生窗口按钮互相遮挡。
- 右侧 Agent 面板可以收起 / 展开，团队文件可以直接拖入 Agent 输入区作为待解读文件。
- 上传按钮使用分裂按钮：主按钮上传文件，右侧展开菜单可选择上传文件或上传文件夹。

## 拖拽规则

- 从系统拖入文件到当前目录空白区域：上传到当前目录。
- 从系统拖入文件到某个文件夹卡片、列表行或左侧树文件夹：直接上传到该文件夹。
- 在团队文件区内部拖动文件或文件夹到目标文件夹：执行远程移动，并清理旧路径下的本地缓存。
- 把团队文件拖到右侧 Agent 面板：加入当前 Agent 会话的待发送文件；文件夹暂不支持直接拖入 Agent，需要拖入具体文件。
- 团队文件拖入 Agent 使用专用 `application/x-proma-team-file` 拖拽类型，并在 Agent 面板捕获阶段处理，避免被普通本地文件拖拽逻辑提前消费。

## 本地与服务器语义

- 服务器文件清单是唯一数据源；客户端不做后台全量同步。
- 本地 `workspace-files/` 是按需缓存，用于降低预览、打开和二次解读的等待时间。
- 删除团队文件时，服务器删除成功后会同步清理对应本地缓存，避免残留文件误导预览状态。
- 移动团队文件时，服务器移动成功后会清理旧路径缓存；目标路径后续按需下载或由上传流程写入。
- 打开文件夹所在位置时展示的是本地缓存目录；如果文件还未缓存，需要先下载或预览一次。
