# Profer v0.12.69

> 对比基准：v0.12.67

这一版从 Proma upstream 同步了三项交互改进：侧栏委派子会话树形折叠、Markdown 目录手动开闭、以及全新的经典/现代界面风格切换。同时补齐了协作委派（collaboration）的完整后端链路，修复了版本更新渠道校验问题。

## 新功能

- **侧栏委派子会话树形折叠** — 通过 `delegate_agent` / `delegate_agents` 创建的协作子会话不再平铺在侧栏，而是嵌套收进母会话下方形成可折叠树。展开/收起箭头 + `已完成/总数` 子会话计数器，置顶/归档母会话时级联同步所有子会话，失败时自动从磁盘恢复真实状态避免不一致
- **Markdown 预览目录手动开闭** — 目录 TOC 改为仅运行时记忆（重启恢复默认展开），移除窄屏自动收起逻辑改为用户手动控制。新增展开/收起按钮，收起后在预览区左侧显示展开入口
- **经典/现代界面风格切换** — 外观设置新增「界面风格」选项（默认现代），经典风保留旧版视觉（圆角阴影卡片），现代风使用独立的侧栏/标签栏/输入区表面 token、扁平 Tab 和 1px 分割线。两种风格共享相同的圆角基准和间距体系
- **协作委派（collaboration）完整后端** — 移植全套 collaboration MCP 工具链路：`delegate_agent` / `delegate_agents` / `wait_for_delegations` / `list_delegations` / `get_delegation_results` / `stop_delegation` / `stop_delegations` / `answer_delegation_question` / `continue_delegation` / `list_available_agent_models` 共 10 个工具，以及 headless runner registry、模型选择验证、阻塞事件处理、委派状态恢复。子会话元数据同步写入 `agent-sessions.json`，侧栏切换可见

## Bug 修复

- **修复版本更新渠道错位** — 默认构建命令 `dist:win` 误注入 oss 构建目标导致运行时查询 GitHub Releases 而非国内更新服务器，v0.12.67 用户无法自动收到更新。现统一走自建更新服务器，保留 `dist:win-github` / `dist:win-commercial` 作为显式选择
- **Channel/Provider 稳定性修复** — 同步 upstream 的 DeepSeek chat/agent 端点分离、模型配置路由加固、provider 连通性测试收紧、oss 渠道管理隔离等多项修复
