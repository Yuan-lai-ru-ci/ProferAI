## 快速开始

如果你是第一次接触 Profer，这部分带你从零走完：下载 → 注册 → 配置 → 第一次对话。整个过程大约 10-15 分钟。

### 1. 下载与安装

从 [GitHub Releases](https://github.com/Yuan-lai-ru-ci/Profer/releases) 下载最新安装包。

- **Windows**：下载 `Profer Setup x.x.x.exe`，双击安装。Defender 可能提示"未知发布者"（应用尚未签名，不影响安全），点击"更多信息 → 仍要运行"即可。
- **macOS**：下载 `.dmg`（Apple Silicon 选 arm64，Intel 选 x64），双击挂载后拖入 Applications。

首次启动会自动环境检查。Windows 用户会被引导安装 Git、Node.js 等依赖，跟随界面操作即可——**务必勾选 "Add to PATH"**，否则 Agent 没法用。

### 2. 注册与登录

Profer 同时支持**个人使用**和**团队协作**。个人使用时所有数据存本地，无需注册，直接跳到"配置 AI 渠道"即可。

如果你收到了团队管理员的邀请码：

1. 点击左侧边栏底部的「登录」按钮，或进入 设置 → 团队
2. 在弹窗中切换到「注册」标签页
3. 填写邮箱、显示名称、密码（至少 8 位，需含大小写字母和数字）和邀请码
4. 点击「注册」，成功后自动加入对应工作区，侧边栏会出现团队工作区列表

已有账号则在「登录」标签页输入邮箱密码。登录后自动同步所有已加入的团队工作区。

> 如果你是团队管理员，第一个管理员账号需在服务器端创建（参考 `server/README.md`），之后即可在客户端为成员生成邀请码。

### 3. 配置 AI 渠道

无论个人还是团队，都需要至少一个 AI 渠道才能对话。

1. 进入 **设置 → 渠道**，点击「添加渠道」
2. 选择**供应商类型**。推荐优先选标有「国内·推荐」的供应商（DeepSeek、通义千问、Kimi、智谱等），网络更稳定
3. 填入 **API Key**（从对应供应商开放平台获取）
4. Base URL 和模型列表一般自动填充，无需修改
5. 点击「创建」

可添加多个渠道，在不同模型间切换。

> **Agent 模式需要 Anthropic 协议兼容渠道**。DeepSeek、Kimi Coding Plan、GLM Coding Plan 均支持 Agent。OpenAI GPT 系列和智谱非 Coding Plan 套餐仅支持 Chat。

### 4. 第一次 Agent 对话

Profer 有 **Chat** 和 **Agent** 两种模式，顶部标签栏切换：

- **Chat**：问答、翻译、润色、轻量代码讨论
- **Agent**：读写文件、执行命令、多步骤工作流——它会主动使用工具

第一次上手建议直接切到 Agent 模式，给一个简单但完整的任务，比如：

> 在当前工作区文件夹下创建 `hello.md`，写一段项目介绍模板，包含项目名称、目标、技术栈。

然后观察 Agent 如何规划、征求确认、执行、输出结果。右侧面板会显示 Agent 操作的文件，左下角显示当前可用的 Skills 和 MCP 工具。

### 5. 常用操作速览

| 操作 | 怎么做 |
|------|--------|
| 创建新会话 | 左侧会话列表项目旁 `+` 按钮 |
| 切换工作区 | 左侧下拉菜单选择个人/团队工作区 |
| 添加文件给 Agent | 拖拽文件到输入框，或用 `@` 引用 |
| 查看/管理 Skills | 左下角 Skills 管理区域 |
| 邀请团队成员 | 团队工作区设置 → 生成邀请码 → 发送 |
| 团队文件管理 | 切换到团队工作区，右侧文件面板上传/下载/拖拽 |
| 切换外观 | 设置 → 外观 → 亮色/暗色/跟随系统 |

## 理解 Agent

如果你只用过 ChatGPT 这类聊天 AI，Agent 模式可能会让你感到陌生。这一章解释 Agent 到底是什么、为什么它和 Chat 有本质区别、以及如何正确地使用它。

### Agent 和 Chat 到底差在哪？

Chat 模式就是"你问它答"。你发一条消息，模型回一条消息，对话结束。它不能操作你的文件、不能执行命令、不能分步骤完成一个复杂目标。

Agent 模式完全不同。Agent 拿到你的任务后，会自己拆解步骤、选择合适的工具（读写文件、搜索代码、执行命令、调用 Skills 和 MCP）、执行每一步并根据结果调整下一步计划。你可以把它想象成一个"有手有脑的助手"——脑是 AI 模型，手是这些工具。

举个例子。你说"帮我把这个 Excel 里的销售数据做个分析报告，输出成 Markdown"：

- Chat 模式：它只能告诉你"你可以用 Pandas 读取 Excel，然后……"，你得自己动手
- Agent 模式：它自己读取 Excel → 分析数据 → 计算指标 → 写报告 → 保存到你指定的路径。你只需要在关键步骤确认

**Agent 的核心能力是"实现"，而不仅是"建议"。** 这就是为什么 Agent 是这一轮 AI 最重要的产品形态——它从"告诉你"进化到了"帮你做"。

### 什么是好的提示词？

人类跟人类说话时，我们借用了环境作为上下文，省略了很多东西。比如你跟同事说"帮我把那个报表做一下给小李"——这在人的语境里能理解，但对 Agent 来说是个灾难。

好的 Agent 提示词需要**完整、明确、有上下文**。上面那句话转换过来应该是：

> 在当前工作区文件夹下有个 @sales.xlsx，这是我们本月的销售报表。核心产品是 a、b、c，成本数据在 @cost.xlsx 的 sheet2。我希望你先读一下这两个文档，然后计算各产品的销售毛利率，把结果写到 @sales.xlsx 的 sheet1。关于格式我们有个模板在 @template.md，你先参照整理。如果数据有缺漏，明确指出缺了哪些，我来补充。有任何不确定的部分都停下来跟我确认。完成后给我一份分析方法的总结。

是的，就是可能需要这么长。长到让人厌烦的程度，就是一个好提示词。如果你觉得打字太累，Profer 支持语音输入，效果还不错。

更进一步——如果你发现自己反复做类似的工作，把它沉淀成 Skills（第四章会讲），下次一句话就能触发整个流程。

### 会话管理：一件事一个会话

**不要在一个会话里持续交互几十次甚至上百次。** 这是新手最常犯的错误。

一个会话只处理一个简短的任务。原因很简单：上下文越长，每次请求消耗的 Token 越多（成本越高）、响应越慢、模型能力越弱。干净的短上下文 → 更快、更省钱、更准确。

当然也不用过度追求——比如你昨天做了一次数据分析，今天数据更新了想重新跑一遍，在昨天的会话下说"数据更新了，按上次的方案重新分析"是完全合理的。

如果你有大型任务一个会话做不完，让 Agent 把进度、设计和踩坑记录写到 `.context/` 目录下，然后开新会话引用这些文件继续。Profer 支持用 `&` 符号引用历史会话。

## 文件系统 —— Agent 的双手

Agent 能做事的核心原因是它能**读写文件**。文件本身就是上下文——而且是经过你精心组织和筛选过的优质上下文。所以理解 Profer 的文件设计，是用好 Agent 的关键。

### 工作区文件 vs 会话文件

Profer 右侧文件面板有三个 Tab，你需要理解最重要的区别：

**工作区文件**：适合那些"这一类任务都可能用到"的文件。比如你开发 Profer 时，Profer 的项目代码就是工作区文件——每次开新会话开发新功能，Agent 都能访问这些代码。或者你跟 Agent 一起做的长期规划文档，未来某天还会用到，也应该放工作区。

**会话文件**：适合"只跟当前这一次任务有关"的临时文件。比如你要 Agent 帮你设计一个新功能的方案，这个设计方案只适用于当前会话，下次就用不上了。拖拽文件到输入框默认添加到会话文件夹。

**附加文件/文件夹**：文件已经在你电脑上，可能很大很多，不想重复占用磁盘空间？用附加功能。比如你可以附加 Obsidian 的笔记目录，Profer 就能直接读写这些文件——不需要导入导出。

### .context 目录

你可能会在工作区和会话文件夹下看到一个 `.context` 目录。这是给 Agent 自己用的——Agent 会在这里写 note、todo、plan 等中间产物。这是 Agent 的"工作台"，你不用手动管理，但如果你有明确需求，也可以让 Agent 按你的要求写入这个目录。

### 文件就是上下文

你组织的文件越干净、越合理，Agent 输出就越快、越好、越省钱。**每一次微小的文件组织决策，都是在对 Agent 的上下文做迭代优化。**

不需要第一次就做到完美。文件可以随时删改。当你意识到"这个文件下次还会用"时，把它移到工作区；当你发现"这个文件已经没用了"时，删掉它。这就是上下文组织的迭代——做得越多，Agent 表现越好。

### 进阶：用文件构建个人数据库

你可能会问：能不能让 Agent 帮我"记住"一些信息，跨会话使用？

文件就是最好的"数据库"。比如你是一位面试官，可以让 Agent 把每份简历的关键信息提炼成 JSONL 格式，存到工作区下的某个文件夹。积累多了之后，你就可以在新会话里问："上次那个做后端的人跟这次这个比，谁更匹配？"或者"我最近三个月面试的人里，哪些人提到了 Rust？"

本质上，你在用文件构建一个你自己的、Agent 可以理解和查询的知识库。这比任何云端记忆方案都可靠——数据在你电脑上，你完全掌控。

## Skills 和 MCP —— Agent 的技能包

Skills 和 MCP 是给 Agent "装备"能力的两种方式。理解它们不复杂：Skills 是给 Agent 的"工作流程手册"，MCP 是给 Agent 的"外部服务接口"。

### Skills 是什么？

Skills 的核心是把一类工作的流程、判断标准和注意事项写成结构化的提示词。Agent 在处理对应任务时自动加载——不需要你每次都说一遍"先做 A，然后检查 B，如果 C 就做 D"。

比如 Profer 内置了 `skill-creator`——你可以直接跟 Agent 说"帮我把刚才的工作流创建为 Skill"，Agent 会跟你确认流程细节，然后生成一个可复用的 Skill。下次一句话触发。

**创建 Skills 的难度并不比发一条微博更难。** 门槛低到无法想象，所以你要做的是更谨慎和更真实——只有经过实际检验的、确实反复使用的流程，才值得变成 Skill。

### 如何发现和安装 Skills？

互联网上已经存在大量 Skills。Profer 内置了 `find-skills`，你可以在 Agent 模式下直接说："帮我找一些可以辅助生物学研究的 Skills"——Agent 会搜索、展示选项、等你确认后自动安装。

Profer 还支持 **Skills 市场**——团队成员可以共享 Skills。一个人创建的好用工作流，全团队都能受益。

### 不要装太多 Skills

Erlich Liu 在 Proma 教程里说过，市面上绝大部分 Skills 并不好，只是为了存在而存在。Karpathy 的开发 Skills 和 YC CEO 的 Skills 名气大，但自媒体要的是炒作，你付的是 Token 费。你和一个解决你真实问题的普通 Skill 之间没有不可逾越的鸿沟——实际上你自己做的可能更好用。

### MCP 是什么？

MCP（Model Context Protocol）是连接外部服务的方式。比如你安装了飞书 MCP，Agent 就能操作你的飞书文档、表格、日历；装了 Gmail MCP，就能帮你读写邮件。

安装 MCP 同样很简单——在 Agent 模式下说"帮我安装飞书 MCP"，Agent 会引导你完成授权配置，重启 Profer 就生效了。

MCP 本身没什么复杂的。你只需要知道它存在，当你有"Agent 要是能直接操作 XX 就好了"这种想法时，回来让 Agent 帮你搜索和安装就是了。

## 团队协作

这是 Profer 和 Proma 最大的区别——也是我投入最多精力的部分。

### 工作区概念

Profer 的工作区分**个人工作区**和**团队工作区**。个人工作区的所有数据在你本地，团队工作区的数据在服务器上，成员之间实时同步。

这个设计基于一个简单的前提：你可能同时有"自己的工作"和"团队的工作"。个人工作区是私密的——你的实验性项目、个人笔记、自己用的 Skills；团队工作区是共享的——团队文件、共享 Skills、协作项目。

左侧边栏顶部可以切换工作区。登录后自动同步所有你已加入的团队工作区。

### 团队文件共享

切换到团队工作区后，右侧文件面板就是团队的共享文件空间：

- **上传/下载**：拖拽或点击上传，团队成员立即可见
- **权限管理**：上传者和管理员可以重命名、删除自己的文件
- **文件预览**：支持 Markdown、代码、图片等格式的标签页预览
- **重命名保护**：自动保留文件扩展名，防止误改格式导致文件打不开

### 活动动态与通知

团队工作区有**活动动态（Activity Feed）**——谁上传了文件、谁创建了新会话、谁分享了 Skills，所有成员都能看到。同时支持**桌面通知**，重要事件不会错过。

### 邀请成员

团队管理员在团队工作区设置中生成邀请码，发送给成员。成员在注册时填入邀请码即可加入。一个成员可以加入多个团队，在侧边栏自由切换。

## 知识库 —— 让 Agent 读论文

这是我作为科研工作者的刚需。Profer 内置了完整的知识库系统，可以把论文导入、解析、然后用自然语言搜索。

### 怎么用？

1. 在 Agent 模式下，直接拖入一篇 PDF 论文，或告诉 Agent 一个 arXiv ID（比如 `2401.12345`）
2. Agent 会调用 MinerU 将论文解析为结构化 Markdown——保留公式、表格和章节结构
3. 解析后的论文存入知识库，自动分块并生成向量
4. 之后你就可以用自然语言搜索了："这篇论文的实验部分用的是哪个数据集？"或者"方法论部分提到的那个基线模型是什么？"

### 为什么不用 ChatGPT 上传 PDF？

ChatGPT 上传 PDF 是单次对话级别的——这次对话结束，下次还要重新上传。Profer 的知识库是持久化的——导入一次，永久可搜索。而且 Profer 是在你本地或你团队的服务器上管理这些数据，不存在"论文被用来训练模型"的隐私风险。

### 科研工作流示例

一个典型的研究场景：你想了解某个方向的最新进展。

1. 在 arXiv 上找到 5 篇相关论文，把 arXiv ID 告诉 Agent
2. Agent 逐篇导入并解析
3. 你问 Agent："这 5 篇论文的方法有什么共同点和差异？给我做个对比表"
4. Agent 搜索知识库、提取关键信息、生成对比报告
5. 把报告保存到工作区 `.context/` 下，下次还能用

整个过程，你只需要提供论文 ID 和提出问题。Agent 负责剩下的。

## 自动化任务 —— 让 Agent 自己跑

大概两个月前我仍然抵触做自动任务——那时候大部分"自动化需求"都是伪需求，比如定时抓新闻。但今天不一样了。随着使用 Agent 的时间越来越长，真正的自动化场景开始浮现。

### 什么是好的自动化任务？

好的自动化任务不是"定时提醒你喝水"。它是有真实业务价值的、反复发生的、可以无人值守完成的流程。

比如 Profer 自己的自动化：
- 每天自动审计生产日志，检查异常
- 每周汇总用户反馈，生成周报
- 追踪上游 Proma changelog，判断是否有需要移植的改动

如果你发现自己和 Agent 协作完成一件事后，心想"这做得还不错，下次还要做"——这就是最好的自动化信号。

### 如何创建？

Profer 内置了 `automation` Skill。在 Agent 模式下直接说你的需求，比如"帮我每天早上 9 点检查一遍团队服务器的健康状态，结果通过飞书推给我"，Agent 会帮你创建并试运行。

支持四种调度：
- **固定间隔**：每 N 分钟执行一次
- **每天定点**：比如每天 09:00
- **每周定点**：比如每周一 09:00
- **每月定点**：比如每月 1 号

创建后可以在侧边栏的"自动任务"管理中查看运行记录、调整频率、暂停或删除。

自动化结果也支持通过飞书推送——不在电脑前也能收到。

## 任务图 —— 可视化管理复杂工作

当你给 Agent 一个复杂任务时，Agent 会自动把它拆成多个子任务。Profer 的**任务图**就是把这些子任务之间的关系可视化出来。

### 三种关系

- **依赖（dependsOn）**：任务 B 必须等任务 A 完成才能开始
- **分叉（forkFrom）**：原来的方向走不通了，从某个任务分叉出新方向
- **放弃（abandon）**：某个方向被放弃，但保留记录（不会静默消失）

你可以在项目图谱中看到这些任务节点的实时状态——哪些在进行、哪些已完成、哪些被放弃。点击节点可以跳转到对应的会话或文件。

任务图的核心设计理念是"靠 Agent 自律"——Agent 自己管理任务拆分和状态更新，你只需要在关键节点确认。这跟传统的项目管理工具不同：你不是在手动拖拽任务卡片，而是在观察 Agent 如何规划和执行。

## 飞书集成 —— 手机上的 Agent

飞书是目前 Profer 支持最好的远程渠道，体验远超微信和钉钉。

### 配置

在设置 → 飞书中扫码完成配置。整个过程 3-5 分钟。

### 手机端体验

配置完成后，你可以在飞书中给 Profer Agent 发消息——指派任务、查询状态、接收结果。飞书支持流式卡片，对话体验不比桌面版差太多。大概能覆盖 80% 的手机端 Agent 需求。

### 飞书 CLI

滑到飞书配置页底部安装 CLI，完成后 Agent 可以直接操作你的飞书文档、表格、日历和会议。这是我个人使用频率最高的集成之一——不在电脑前时，用手机通过飞书给 Agent 指派任务，Agent 执行完后把结果发回飞书或邮件。

### 自动任务推送

自动化任务完成后，结果可以推送到飞书——做到真正的"无人值守，手机收结果"。

## 安全与设置

### 设备管理

设置 → 设备管理可以查看所有登录过你账号的设备。如果发现可疑设备，一键远程登出。Profer 也支持设备上限保护——满额时自动弹窗，你可以选择踢掉一台旧设备再登录。

### 客户端加密

Profer 使用 **AES-256-GCM** 加密存储你的 API Key 和 Token。加密密钥优先使用操作系统安全存储（Windows DPAPI / macOS Keychain），降级时自动切换到软件加密方案。简单说：即使有人拿到你的 Profer 数据文件，没有你的操作系统登录密码也解不开。

## 更进一步

Profer 还在飞快进化。对于已经读到这里的你，最后几点建议：

**用 `/coach`**：在 Agent 模式下输入 `/coach`，Profer Coach 会帮你分析使用习惯，给出优化建议。很多时候你不是缺功能，而是不知道某个功能已经存在。

**每次迭代一点点**：不管用任何 Agent 产品，尝试这次比上次多说一句话——多给一个约束条件，多描述一点上下文，多让 Agent 输出一个总结。一个月后你会发现完全不一样。

**把好用的工作流变成 Skills**：当你在某个会话里完成一项满意的任务后，直接跟 Agent 说"帮我把刚才的工作流程创建为 Skill"。这就是你对 Agent 的"编程"——不需要写代码，只需要描述你做对了什么。

**保持简单**：不需要安装几十个 Skills 或 MCP，不需要追求最复杂的自动化。解决真实的问题，每次迭代一点点，就够了。
